Introdução

O capítulo anterior introduziu o Bootstrap — um framework CSS que encapsula grelhas, componentes e utilitários em classes prontas a usar. O que ficou por dizer é que o Bootstrap é, ele próprio, escrito em SASS. As variáveis que definem cores, tamanhos e breakpoints são variáveis SASS; os mixins que geram as classes responsivas do grid são mixins SASS. Quem domina SASS compreende o Bootstrap de dentro para fora.

Este capítulo introduz o SASS como linguagem independente — aplicável a qualquer projecto, com ou sem framework. O ponto de partida é o problema que o SASS resolve: o CSS, escrito sem estrutura, não escala.

7.1 O Problema do CSS a Escala

7.1.1 Valores repetidos

Num projecto real, a mesma cor primária aparece em dezenas de sítios: no botão, no link, no header, no destaque de formulário. Se o cliente pedir para alterar a cor, cada ocorrência tem de ser encontrada e editada manualmente:

/* Num projecto sem SASS — a mesma cor em seis lugares */
.btn              { background: #5f4fb8; }
.link             { color: #5f4fb8; }
.header           { border-bottom: 2px solid #5f4fb8; }
.badge            { background: #5f4fb8; }
.form-control:focus { border-color: #5f4fb8; }
.pagination .active { background: #5f4fb8; }

Um find-and-replace não é solução — pode atingir a mesma string noutros contextos. O SASS resolve isto com uma variável: altera-se num único lugar e a mudança propaga-se para todo o projecto.

7.1.2 Falta de estrutura

O HTML organiza-se hierarquicamente — um .card contém um .card__titulo que contém um h3. No CSS, essa hierarquia desaparece: cada selector é escrito de forma plana, sem relação visual com a estrutura HTML:

/* CSS plano — a hierarquia do HTML é invisível */
.card                     { background: white; border-radius: 8px; }
.card .card__titulo       { font-size: 1.25rem; font-weight: bold; }
.card .card__titulo:hover { color: #5f4fb8; }
.card .card__corpo        { padding: 1rem; }
.card .card__footer       { font-size: 0.85rem; color: #666; }

À medida que o projecto cresce, este CSS disperso torna-se difícil de ler e de associar mentalmente à estrutura HTML correspondente.

7.1.3 Manutenção difícil

Sem módulos, um projecto CSS de dimensão média acaba num único ficheiro com centenas ou milhares de linhas. Alterar um componente implica saber onde começa e termina o seu CSS, num ficheiro sem divisões claras. O resultado é o que os programadores designam por CSS soup: funciona, mas ninguém quer tocar.

7.2 O que é o SASS

7.2.1 Pré-processador CSS

O SASS (Syntactically Awesome Style Sheets) é um pré-processador CSS — uma linguagem que estende o CSS com variáveis, nesting, mixins, funções e módulos, e que compila para CSS válido antes de chegar ao browser. Isto é, desenvolve-se o projecto em SASS e há um processo automático (compilador) que gera o CSS.

Ficheiro .scss  →  Compilador SASS  →  Ficheiro .css  →  Browser

Um browser não funciona com SASS, mas apenas com CSS. O browser nunca vê código SASS. O que é servido é CSS puro — exactamente o mesmo que seria escrito à mão, mas gerado a partir de uma fonte mais estruturada e mais fácil de manter.

Dart Sass — a implementação actual

A implementação oficial e activa do SASS é o Dart Sass. Versões anteriores (LibSass, Ruby Sass) estão descontinuadas. O comando sass no terminal e a extensão Live Sass Compiler do VS Code usam Dart Sass.

7.2.2 Sintaxe SCSS

O SASS tem duas sintaxes. A mais usada é a SCSS (ficheiros .scss), que é um superset do CSS — qualquer CSS válido é também SCSS válido. A sintaxe alternativa .sass usa indentação em vez de chavetas, mas raramente é adoptada em projectos novos.

/* Isto é SCSS válido */
.btn {
  background: #5f4fb8;
  color: white;
  padding: 0.5rem 1rem;

  &:hover {
    background: #4a3d9a;  /* nesting — ver secção Nesting */
  }
}

O compilador transforma o bloco acima no CSS equivalente:

/* CSS compilado */
.btn { background: #5f4fb8; color: white; padding: 0.5rem 1rem; }
.btn:hover { background: #4a3d9a; }

7.2.3 Compilação no fluxo de trabalho

Em desenvolvimento, o SASS compila automaticamente a cada gravação de ficheiro. As ferramentas mais comuns:

Ferramenta Como funciona
VS Code — Live Sass Compiler Extensão que monitoriza .scss e compila para .css a cada gravação
CLI Dart Sass sass --watch scss/main.scss css/main.css via terminal
WebStorm — File Watchers Configuração nativa que executa o compilador automaticamente

7.3 Variáveis

7.3.1 Declaração e uso

Uma variável SASS guarda um valor que pode ser reutilizado em qualquer parte do projecto. As variáveis são escritas num ficheiro separado, geralmente com o nome _variables.scss. A sintaxe é $nome: valor:

/* Exemplo de variáveis definidas no ficheiro _variables.scss */
$cor-primaria:  #5f4fb8;
$cor-texto:     #1a1a1a;
$fonte-base:    'Segoe UI', Arial, sans-serif;
$tamanho-base:  16px;
$raio:          8px;
$espaco-md:     1rem;

/* As variáveis usam-se dentro de outros ficheiros. P. ex. em _components.scss */
.btn {
  background:    $cor-primaria;
  color:         white;
  font-family:   $fonte-base;
  font-size:     $tamanho-base;
  padding:       $espaco-md 2rem;
  border-radius: $raio;
}

.link { color: $cor-primaria; }
/* O CSS compilado fica assim: */
.btn {
  background: #5f4fb8; color: white;
  font-family: 'Segoe UI', Arial, sans-serif;
  font-size: 16px; padding: 1rem 2rem; border-radius: 8px;
}
.link { color: #5f4fb8; }

Para alterar a cor primária de todo o projecto, basta editar uma linha em _variables.scss.

7.3.2 Tipos de valores

As variáveis SASS aceitam qualquer valor CSS, e também tipos próprios:

Tipo Exemplos
Cor #5f4fb8, rgb(95, 79, 184), hsl(249, 40%, 51%)
Número com unidade 16px, 1rem, 100%, 0.5s
String 'Segoe UI', bold, center
Booleano true, false
Lista 'Segoe UI', Arial, sans-serif
Mapa (sm: 576px, md: 768px, lg: 992px)
Null null — a propriedade é omitida do output

7.3.3 Âmbito — global vs local

Uma variável declarada fora de qualquer bloco é global — acessível em todo o ficheiro. Declarada dentro de um bloco, é local — acessível apenas nesse bloco e nos seus descendentes:

$cor-primaria: #5f4fb8;   /* global */

.card {
  $raio-interno: 6px;      /* local — só existe dentro de .card */

  border-radius: $raio-interno;  /* ✓ dentro do âmbito */
  color: $cor-primaria;          /* ✓ variável global */
}

.btn {
  border-radius: $raio-interno;  /* ✗ fora do âmbito — erro de compilação */
}

7.3.4 Variáveis SASS vs Custom Properties CSS

Pode surgir uma questão legítima: se o problema dos valores repetidos é tão evidente, o CSS não o terá resolvido por conta própria? A resposta é sim — o CSS tem o seu próprio sistema de variáveis, as custom properties (propriedades personalizadas), desenvolvido de forma completamente independente do SASS. Quem trabalha com SASS depara-se portanto com dois mecanismos de variáveis distintos, e convém perceber as diferenças.

As custom properties CSS escrevem-se com dois traços antes do nome e declaram-se habitualmente no selector :root (que corresponde ao elemento <html>, o topo da árvore da página), para ficarem disponíveis em todo o documento. Para ler o valor, usa-se a função var():

/* Custom properties CSS — variáveis nativas da linguagem */
:root {
  --cor-primaria: #5f4fb8;
  --tamanho-fonte: 16px;
}

.btn {
  background: var(--cor-primaria);   /* acede-se ao valor com var() */
  font-size:  var(--tamanho-fonte);
}

À primeira vista, o resultado é semelhante ao das variáveis SASS: um valor centralizado, reutilizável em todo o documento. A diferença fundamental está em quando cada mecanismo actua:

  • As variáveis SASS ($var) desaparecem na compilação — o compilador substitui cada $cor-primaria pelo valor concreto #5f4fb8 antes de gerar o CSS. O browser nunca chega a ver a variável SASS.

  • As custom properties CSS (--var) persistem no browser — o browser recebe o ficheiro CSS com --cor-primaria: #5f4fb8 declarado e resolve a função var() em tempo de execução (à medida que a página é apresentada e pode até ser alterada enquanto a página está aberta).

Esta diferença tem consequências práticas:

Característica SASS $var CSS --var
Existe no CSS compilado? Não Sim
Acessível em JavaScript? Não Sim (getComputedStyle)
Pode mudar em tempo de execução? Não Sim (media queries, JS)
Funciona em calc()? Não (usa operadores SASS) Sim
Ideal para Valores de design estáticos Temas dinâmicos, adaptação responsiva
Usar os dois em conjunto

Uma boa prática é definir valores em variáveis SASS e expô-los como custom properties no :root, combinando o melhor de ambos:

$cor-primaria: #5f4fb8;

:root {
  --cor-primaria: #{$cor-primaria};  /* interpolação: #{} converte para string */
}

O JavaScript (a linguagem de programação dos browsers, que permite modificar a página depois de carregada) pode ler e alterar --cor-primaria em tempo de execução; o SASS usa $cor-primaria para cálculos durante a compilação.

7.4 Nesting

7.4.1 Hierarquia no CSS

O nesting permite escrever regras CSS aninhadas, espelhando a hierarquia do HTML. O compilador transforma cada nível aninhado num selector descendente:

/* SCSS */
.card {
  background: white;
  border-radius: $raio;

  .card__titulo {
    font-size: 1.25rem;
    font-weight: bold;
  }

  .card__corpo {
    padding: $espaco-md;
  }
}
/* CSS compilado */
.card { background: white; border-radius: 8px; }
.card .card__titulo { font-size: 1.25rem; font-weight: bold; }
.card .card__corpo  { padding: 1rem; }

A relação visual entre .card e os seus elementos é imediata no SCSS; no CSS plano equivalente, tem de ser inferida pelo leitor.

7.4.2 O & — referência ao selector pai

O & representa o selector do bloco pai. Permite escrever pseudo-classes, pseudo-elementos e modificadores no contexto do componente:

.btn {
  background: $cor-primaria;
  transition: background 0.2s ease, transform 0.1s ease;

  &:hover        { background: color.adjust($cor-primaria, $lightness: -10%); }
  &:active       { transform: translateY(1px); }
  &:focus-visible {
    outline: 3px solid $cor-primaria;
    outline-offset: 2px;
  }

  /* Modificadores BEM */
  &--grande   { font-size: 1.125rem; padding: 0.75rem 2rem; }
  &--contorno {
    background: transparent;
    border: 2px solid $cor-primaria;
    color: $cor-primaria;
  }
}
/* CSS compilado */
.btn { background: #5f4fb8; transition: background 0.2s ease, transform 0.1s ease; }
.btn:hover         { background: #4a3d9a; }
.btn:active        { transform: translateY(1px); }
.btn:focus-visible { outline: 3px solid #5f4fb8; outline-offset: 2px; }
.btn--grande       { font-size: 1.125rem; padding: 0.75rem 2rem; }
.btn--contorno     { background: transparent; border: 2px solid #5f4fb8; color: #5f4fb8; }

7.4.3 A regra dos 3 níveis

O nesting em excesso gera selectores CSS excessivamente específicos e difíceis de sobrepor:

/* ✗ Nesting excessivo — 5 níveis */
.header {
  .nav {
    .nav__lista {
      .nav__item {
        .nav__link { color: white; }
        /* compila para: .header .nav .nav__lista .nav__item .nav__link */
      }
    }
  }
}

A regra prática é não ultrapassar 3 níveis de nesting. Se o quarto nível for necessário, é geralmente sinal de que o componente deve ser refactorado em blocos independentes.

/* ✓ Máximo 3 níveis */
.nav {
  &__lista { list-style: none; display: flex; }
  &__item  { margin: 0 0.5rem; }
  &__link  {
    color: white;
    &:hover { color: $cor-primaria; }  /* 3.º nível — aceitável */
  }
}

7.5 Partials e Módulos

7.5.1 Partials

Um partial é um ficheiro SCSS cujo nome começa com _ (underscore). O compilador nunca o processa de forma autónoma — só o inclui quando é importado por outro ficheiro. Esta convenção permite dividir o CSS em ficheiros temáticos sem gerar um .css individual para cada um:

scss/
├── _variables.scss   ← tokens de design (cores, tipografia, espaçamentos)
├── _base.scss        ← reset e estilos globais
├── _components.scss  ← botões, cards, mensagens
├── _layout.scss      ← header, footer, grelha
└── main.scss         ← ponto de entrada — importa todos os anteriores

Só o main.scss (sem underscore) compila para css/main.css. Os partials são peças que o main.scss agrega.

7.5.2 @import vs @use

Existem duas formas de incluir um partial: @import (método antigo, em extinção) e @use (método moderno, com namespacing):

/* main.scss — método @import */
@import 'variables';    /* todas as variáveis ficam globais */
@import 'base';
@import 'components';
@import 'layout';
/* _layout.scss — método @use para módulos nativos */
@use 'sass:color';      /* acesso via color.adjust() */
@use 'sass:math';       /* acesso via math.div() */
Por que @import está a ser abandonado?

O @import torna todas as variáveis, mixins e funções globais — num projecto grande, torna-se impossível saber de onde vem cada símbolo. O @use resolve isto com namespacing: color.adjust() indica explicitamente que a função vem do módulo color. O @import será removido no Dart Sass 3.0.

7.5.3 Módulos nativos do Dart Sass

O Dart Sass inclui módulos nativos que substituem funções globais em extinção:

@use 'sass:math';   /* math.div(), math.round(), math.floor(), math.ceil() */
@use 'sass:color';  /* color.adjust(), color.mix(), color.scale() */
@use 'sass:string'; /* string.length(), string.to-upper-case() */
@use 'sass:list';   /* list.length(), list.nth() */

7.6 Mixins

7.6.1 @mixin — bloco reutilizável

Um mixin é um bloco de CSS nomeado que pode ser incluído em qualquer selector. É a resposta do SASS ao problema da repetição de padrões:

/* Definição */
@mixin flex-center {
  display: flex;
  align-items: center;
  justify-content: center;
}

/* Uso com @include */
.hero            { @include flex-center; min-height: 100vh; }
.modal           { @include flex-center; position: fixed; inset: 0; }
.cartao__imagem  { @include flex-center; height: 160px; }
/* CSS compilado */
.hero           { display:flex; align-items:center; justify-content:center; min-height:100vh; }
.modal          { display:flex; align-items:center; justify-content:center; position:fixed; inset:0; }
.cartao__imagem { display:flex; align-items:center; justify-content:center; height:160px; }

7.6.2 Mixins com parâmetros

Os mixins aceitam parâmetros, tornando-os equivalentes a funções que geram CSS:

@use 'sass:color';

@mixin btn($cor-fundo: #5f4fb8, $cor-texto: white) {
  background:    $cor-fundo;
  color:         $cor-texto;
  padding:       0.5rem 1.5rem;
  border:        2px solid transparent;
  border-radius: 6px;
  cursor:        pointer;
  font-weight:   600;
  transition:    background 0.2s ease, transform 0.1s ease;

  &:hover  { background: color.adjust($cor-fundo, $lightness: -10%); }
  &:active { transform: translateY(1px); }
}

/* Com valores por defeito */
.btn-primario { @include btn; }

/* Sobrepondo parâmetros */
.btn-sucesso  { @include btn(#198754); }
.btn-perigo   { @include btn(#dc3545); }
.btn-escuro   { @include btn(#212529, white); border-color: #495057; }

7.6.3 Mixins com @content

O @content permite passar um bloco de CSS para dentro de um mixin — útil para encapsular media queries repetidas:

@mixin responde($bp) {
  @if $bp == sm { @media (min-width: 576px) { @content; } }
  @if $bp == md { @media (min-width: 768px) { @content; } }
  @if $bp == lg { @media (min-width: 992px) { @content; } }
}

.grelha {
  grid-template-columns: 1fr;

  @include responde(sm) { grid-template-columns: repeat(2, 1fr); }
  @include responde(md) { grid-template-columns: repeat(3, 1fr); }
}
/* CSS compilado */
.grelha { grid-template-columns: 1fr; }
@media (min-width: 576px) { .grelha { grid-template-columns: repeat(2, 1fr); } }
@media (min-width: 768px) { .grelha { grid-template-columns: repeat(3, 1fr); } }

7.7 Extend e Placeholders

7.7.1 @extend

O @extend faz com que um selector herde todos os estilos de outro. Em vez de duplicar propriedades, o compilador agrupa os selectores numa única regra:

.mensagem {
  padding: 1rem; border-radius: 6px;
  border-left: 4px solid; font-size: 0.9rem; margin-bottom: 1rem;
}

.mensagem-sucesso {
  @extend .mensagem;
  background: #d1e7dd; border-color: #198754; color: #0a3622;
}

.mensagem-erro {
  @extend .mensagem;
  background: #f8d7da; border-color: #dc3545; color: #58151c;
}
/* CSS compilado — selectores agrupados */
.mensagem, .mensagem-sucesso, .mensagem-erro {
  padding: 1rem; border-radius: 6px; border-left: 4px solid;
  font-size: 0.9rem; margin-bottom: 1rem;
}
.mensagem-sucesso { background: #d1e7dd; border-color: #198754; color: #0a3622; }
.mensagem-erro    { background: #f8d7da; border-color: #dc3545; color: #58151c; }

7.7.2 Placeholders %

Um placeholder é um selector especial que começa com %. Não gera CSS por si próprio — só existe para ser estendido. Resolve o problema de @extend .mensagem incluir a classe base no output mesmo que não seja usada directamente no HTML:

/* Placeholder — não gera CSS standalone */
%mensagem-base {
  padding: 1rem; border-radius: 6px;
  border-left: 4px solid; font-size: 0.9rem; margin-bottom: 1rem;
  line-height: 1.5;
}

.mensagem-sucesso {
  @extend %mensagem-base;
  background: #d1e7dd; border-color: #198754; color: #0a3622;
}

.mensagem-aviso {
  @extend %mensagem-base;
  background: #fff3cd; border-color: #ffc107; color: #664d03;
}
/* CSS compilado — sem .mensagem-base a mais */
.mensagem-sucesso, .mensagem-aviso {
  padding: 1rem; border-radius: 6px; border-left: 4px solid;
  font-size: 0.9rem; margin-bottom: 1rem; line-height: 1.5;
}
.mensagem-sucesso { background: #d1e7dd; border-color: #198754; color: #0a3622; }
.mensagem-aviso   { background: #fff3cd; border-color: #ffc107; color: #664d03; }

7.7.3 @extend vs @mixin — quando usar cada um

@extend / %placeholder @mixin
Aceita parâmetros? Não Sim

Pode ser usado em media queries? Não Sim

Regra prática

Usar %placeholder + @extend quando as variantes partilham exactamente o mesmo bloco base e não há parâmetros. Usar @mixin quando há parâmetros ou quando o padrão é usado dentro de media queries.

7.8 Funções e Operadores

7.8.1 Operadores aritméticos

O SASS suporta operações aritméticas directamente nas propriedades CSS:

$tamanho-base: 16px;
$espaco-base:  1rem;

.titulo    { font-size: $tamanho-base * 2;   }  /* 32px  */
.subtitulo { font-size: $tamanho-base * 1.5; }  /* 24px  */
.espaco-lg { margin:    $espaco-base * 2;    }  /* 2rem  */

O operador / foi removido como operador de divisão no Dart Sass moderno, pois conflituava com a sintaxe CSS legítima font: 14px/1.5. A divisão é agora feita com math.div():

@use 'sass:math';

$tamanho-base: 16px;
$espaco-base:  1rem;
$num-colunas:  3;

$tamanho-pequeno: math.div($tamanho-base, 1.25);  /* 12.8px   */
$espaco-sm:       math.div($espaco-base, 2);      /* 0.5rem   */
$largura-coluna:  math.div(100%, $num-colunas);   /* 33.333%  */
Por que o / foi removido?

Em CSS, font: 14px/1.5 é sintaxe válida (tamanho/line-height). Para evitar ambiguidade, o Dart Sass removeu / como operador de divisão. O compilador emite um aviso de extinção se / for usado como divisão sem math.div().

7.8.2 Funções de cor — color.adjust()

$cor-sombra: color.adjust($cor-primaria, $alpha: -0.7);         /* 30% opacidade */

7.9.1 Estrutura do projecto

7.9.3 _components.scss — mixin e placeholder em acção {#sec-07-exemplo-components}

@use 'sass:color';

/* ── Mixin de botão com parâmetros ───────────── */
@mixin btn($cor-fundo: $cor-primaria, $cor-txt: $cor-branco) {
  background:    $cor-fundo;
  color:         $cor-txt;
  padding:       $espaco-sm $espaco-lg;
  border:        2px solid transparent;
  border-radius: $raio;
  cursor:        pointer;
  font-size:     $tamanho-base;
  font-family:   $fonte-base;
  transition:    background 0.2s ease, transform 0.1s ease;

  &:hover  { background: color.adjust($cor-fundo, $lightness: -10%); }
  &:active {
    background: color.adjust($cor-fundo, $lightness: -18%);
    transform: translateY(1px);
  }
}

.btn-primario { @include btn($cor-primaria); }
.btn-escuro   { @include btn($cor-secundaria); }
.btn-contorno {
  @include btn($cor-branco, $cor-primaria);
  border-color: $cor-primaria;
  &:hover { background: color.adjust($cor-primaria, $lightness: 38%); }
}

/* ── Placeholder para mensagens ──────────────── */
%mensagem-base {
  padding:       $espaco-md;
  border-radius: $raio;
  border-left:   4px solid;
  font-size:     $tamanho-pequeno;
  margin-bottom: $espaco-md;
  line-height:   1.5;
}

.mensagem-sucesso {
  @extend %mensagem-base;
  background:   color.adjust(#28a745, $lightness: 48%);
  border-color: #28a745;
  color:        color.adjust(#28a745, $lightness: -12%);
}

.mensagem-erro {
  @extend %mensagem-base;
  background:   color.adjust(#dc3545, $lightness: 38%);
  border-color: #dc3545;
  color:        color.adjust(#dc3545, $lightness: -12%);
}

Neste ficheiro convergem três conceitos do capítulo: o mixin parametrizado gera três variantes de botão a partir de uma única definição; o placeholder evita duplicação nas mensagens sem poluir o CSS com selectores desnecessários; o color.adjust() calcula automaticamente as variantes hover e as cores de estado.

7.9.4 _layout.scss — grelha responsiva {#sec-07-exemplo-layout}

@use 'sass:color';

  @media (min-width: $bp-md) { grid-template-columns: repeat($num-colunas, 1fr); }
}

/* Header */
.header {
  background: $cor-secundaria;
  padding: $espaco-md $espaco-lg;

  &__interior {
    max-width: 1100px; margin: 0 auto;
    display: flex; justify-content: space-between; align-items: center;
  }

    }

7.10 Síntese do Capítulo

O SASS resolve três problemas estruturais do CSS: valores repetidos, falta de hierarquia visível e manutenção difícil. É um pré-processador — compila para CSS antes de chegar ao browser. O browser nunca vê código SASS.

As variáveis ($nome: valor) centralizam tokens de design. Alterar uma variável propaga a mudança para todo o projecto. Distinguem-se das CSS Custom Properties: as variáveis SASS desaparecem na compilação; as --var persistem no browser e podem ser alteradas em tempo de execução por JavaScript.

O nesting espelha a hierarquia do HTML no CSS. O & referencia o selector pai, permitindo escrever pseudo-classes, pseudo-elementos e modificadores BEM no contexto do componente. A regra dos 3 níveis evita selectores excessivamente específicos.

Os partials (_ficheiro.scss) dividem o CSS em módulos temáticos. Só o ponto de entrada (sem underscore) compila para .css. O @use é o método moderno de inclusão — com namespacing, substitui o @import em extinção.

Os mixins (@mixin / @include) encapsulam padrões CSS reutilizáveis com parâmetros. O extend e os placeholders (%) reutilizam blocos base sem parâmetros, agrupando selectores no CSS compilado.

As funções dos módulos sass:math e sass:color permitem cálculos e manipulações de cor em tempo de compilação. math.div() substitui o operador /, removido para evitar conflito com a sintaxe CSS de font.

## Introdução {#sec-07-intro .unnumbered}

O capítulo anterior introduziu o Bootstrap — um framework CSS que encapsula
grelhas, componentes e utilitários em classes prontas a usar. O que ficou
por dizer é que o Bootstrap é, ele próprio, **escrito em SASS**. As variáveis
que definem cores, tamanhos e breakpoints são variáveis SASS; os mixins que
geram as classes responsivas do grid são mixins SASS. Quem domina SASS
compreende o Bootstrap de dentro para fora.

Este capítulo introduz o SASS como linguagem independente — aplicável a
qualquer projecto, com ou sem framework. O ponto de partida é o problema que
o SASS resolve: o CSS, escrito sem estrutura, não escala.

---

## O Problema do CSS a Escala {#sec-07-problema}

### Valores repetidos {#sec-07-problema-valores}

Num projecto real, a mesma cor primária aparece em dezenas de sítios: no
botão, no link, no header, no destaque de formulário. Se o cliente pedir para
alterar a cor, cada ocorrência tem de ser encontrada e editada manualmente:

```css
/* Num projecto sem SASS — a mesma cor em seis lugares */
.btn              { background: #5f4fb8; }
.link             { color: #5f4fb8; }
.header           { border-bottom: 2px solid #5f4fb8; }
.badge            { background: #5f4fb8; }
.form-control:focus { border-color: #5f4fb8; }
.pagination .active { background: #5f4fb8; }
```

Um find-and-replace não é solução — pode atingir a mesma string noutros
contextos. O SASS resolve isto com uma **variável**: altera-se num único lugar
e a mudança propaga-se para todo o projecto.

### Falta de estrutura {#sec-07-problema-estrutura}

O HTML organiza-se hierarquicamente — um `.card` contém um `.card__titulo`
que contém um `h3`. No CSS, essa hierarquia desaparece: cada selector é
escrito de forma plana, sem relação visual com a estrutura HTML:

```css
/* CSS plano — a hierarquia do HTML é invisível */
.card                     { background: white; border-radius: 8px; }
.card .card__titulo       { font-size: 1.25rem; font-weight: bold; }
.card .card__titulo:hover { color: #5f4fb8; }
.card .card__corpo        { padding: 1rem; }
.card .card__footer       { font-size: 0.85rem; color: #666; }
```

À medida que o projecto cresce, este CSS disperso torna-se difícil de ler e de
associar mentalmente à estrutura HTML correspondente.

### Manutenção difícil {#sec-07-problema-manutencao}

Sem módulos, um projecto CSS de dimensão média acaba num único ficheiro com
centenas ou milhares de linhas. Alterar um componente implica saber onde
começa e termina o seu CSS, num ficheiro sem divisões claras. O resultado é
o que os programadores designam por *CSS soup*: funciona, mas ninguém quer
tocar.

---

## O que é o SASS {#sec-07-o-que-e}

### Pré-processador CSS {#sec-07-preprocessador}

O SASS (*Syntactically Awesome Style Sheets*) é um **pré-processador CSS** —
uma linguagem que estende o CSS com variáveis, nesting, mixins, funções e
módulos, e que **compila** para CSS válido antes de chegar ao browser. Isto é,
desenvolve-se o projecto em SASS e há um processo automático (compilador) que
gera o CSS.

```
Ficheiro .scss  →  Compilador SASS  →  Ficheiro .css  →  Browser
```

Um browser não funciona com SASS, mas apenas com CSS. O browser nunca vê
código SASS. O que é servido é CSS puro — exactamente o mesmo que seria
escrito à mão, mas gerado a partir de uma fonte mais estruturada e mais fácil
de manter.

::: {.callout-note}
#### Dart Sass — a implementação actual
A implementação oficial e activa do SASS é o **Dart Sass**. Versões anteriores
(LibSass, Ruby Sass) estão descontinuadas. O comando `sass` no terminal e a
extensão Live Sass Compiler do VS Code usam Dart Sass.
:::

### Sintaxe SCSS {#sec-07-sintaxe}

O SASS tem duas sintaxes. A mais usada é a **SCSS** (ficheiros `.scss`), que
é um *superset* do CSS — qualquer CSS válido é também SCSS válido. A sintaxe
alternativa `.sass` usa indentação em vez de chavetas, mas raramente é adoptada
em projectos novos.

```scss
/* Isto é SCSS válido */
.btn {
  background: #5f4fb8;
  color: white;
  padding: 0.5rem 1rem;

  &:hover {
    background: #4a3d9a;  /* nesting — ver secção Nesting */
  }
}
```

O compilador transforma o bloco acima no CSS equivalente:

```css
/* CSS compilado */
.btn { background: #5f4fb8; color: white; padding: 0.5rem 1rem; }
.btn:hover { background: #4a3d9a; }
```

### Compilação no fluxo de trabalho {#sec-07-compilacao}

Em desenvolvimento, o SASS compila automaticamente a cada gravação de ficheiro.
As ferramentas mais comuns:

| Ferramenta | Como funciona |
|:-----------|:-------------|
| **VS Code — Live Sass Compiler** | Extensão que monitoriza `.scss` e compila para `.css` a cada gravação |
| **CLI Dart Sass** | `sass --watch scss/main.scss css/main.css` via terminal |
| **WebStorm — File Watchers** | Configuração nativa que executa o compilador automaticamente |

## Variáveis {#sec-07-variaveis}

### Declaração e uso {#sec-07-variaveis-declaracao}

Uma variável SASS guarda um valor que pode ser reutilizado em qualquer parte
do projecto. As variáveis são escritas num ficheiro separado, geralmente com
o nome `_variables.scss`. A sintaxe é `$nome: valor`:

```scss
/* Exemplo de variáveis definidas no ficheiro _variables.scss */
$cor-primaria:  #5f4fb8;
$cor-texto:     #1a1a1a;
$fonte-base:    'Segoe UI', Arial, sans-serif;
$tamanho-base:  16px;
$raio:          8px;
$espaco-md:     1rem;

/* As variáveis usam-se dentro de outros ficheiros. P. ex. em _components.scss */
.btn {
  background:    $cor-primaria;
  color:         white;
  font-family:   $fonte-base;
  font-size:     $tamanho-base;
  padding:       $espaco-md 2rem;
  border-radius: $raio;
}

.link { color: $cor-primaria; }
```

```css
/* O CSS compilado fica assim: */
.btn {
  background: #5f4fb8; color: white;
  font-family: 'Segoe UI', Arial, sans-serif;
  font-size: 16px; padding: 1rem 2rem; border-radius: 8px;
}
.link { color: #5f4fb8; }
```

Para alterar a cor primária de todo o projecto, basta editar uma linha em
`_variables.scss`.

### Tipos de valores {#sec-07-variaveis-tipos}

As variáveis SASS aceitam qualquer valor CSS, e também tipos próprios:

| Tipo | Exemplos |
|:-----|:---------|
| Cor | `#5f4fb8`, `rgb(95, 79, 184)`, `hsl(249, 40%, 51%)` |
| Número com unidade | `16px`, `1rem`, `100%`, `0.5s` |
| String | `'Segoe UI'`, `bold`, `center` |
| Booleano | `true`, `false` |
| Lista | `'Segoe UI', Arial, sans-serif` |
| Mapa | `(sm: 576px, md: 768px, lg: 992px)` |
| Null | `null` — a propriedade é omitida do output |

### Âmbito — global vs local {#sec-07-variaveis-escopo}

Uma variável declarada **fora** de qualquer bloco é global — acessível em
todo o ficheiro. Declarada **dentro** de um bloco, é local — acessível apenas
nesse bloco e nos seus descendentes:

```scss
$cor-primaria: #5f4fb8;   /* global */

.card {
  $raio-interno: 6px;      /* local — só existe dentro de .card */

  border-radius: $raio-interno;  /* ✓ dentro do âmbito */
  color: $cor-primaria;          /* ✓ variável global */
}

.btn {
  border-radius: $raio-interno;  /* ✗ fora do âmbito — erro de compilação */
}
```

### Variáveis SASS vs Custom Properties CSS {#sec-07-variaveis-vs-css}

Pode surgir uma questão legítima: se o problema dos valores repetidos é tão
evidente, o CSS não o terá resolvido por conta própria? A resposta é sim —
o **CSS tem o seu próprio sistema de variáveis**, as *custom properties*
(propriedades personalizadas), desenvolvido de forma completamente independente
do SASS. Quem trabalha com SASS depara-se portanto com **dois mecanismos de
variáveis distintos**, e convém perceber as diferenças.

As custom properties CSS escrevem-se com dois traços antes do nome e
declaram-se habitualmente no selector `:root` (que corresponde ao elemento
`<html>`, o topo da árvore da página), para ficarem disponíveis em todo o
documento. Para ler o valor, usa-se a função `var()`:

```css
/* Custom properties CSS — variáveis nativas da linguagem */
:root {
  --cor-primaria: #5f4fb8;
  --tamanho-fonte: 16px;
}

.btn {
  background: var(--cor-primaria);   /* acede-se ao valor com var() */
  font-size:  var(--tamanho-fonte);
}
```

À primeira vista, o resultado é semelhante ao das variáveis SASS: um valor
centralizado, reutilizável em todo o documento. A diferença fundamental está
em **quando** cada mecanismo actua:

- As variáveis SASS (`$var`) **desaparecem na compilação** — o compilador
  substitui cada `$cor-primaria` pelo valor concreto `#5f4fb8` antes de
  gerar o CSS. O browser nunca chega a ver a variável SASS.

- As custom properties CSS (`--var`) **persistem no browser** — o browser
  recebe o ficheiro CSS com `--cor-primaria: #5f4fb8` declarado e resolve
  a função `var()` em tempo de execução (à medida que a página é
  apresentada e pode até ser alterada enquanto a página está aberta).

Esta diferença tem consequências práticas:

| Característica | SASS `$var` | CSS `--var` |
|:---------------|:-----------:|:-----------:|
| Existe no CSS compilado? | Não | Sim |
| Acessível em JavaScript? | Não | Sim (`getComputedStyle`) |
| Pode mudar em tempo de execução? | Não | Sim (media queries, JS) |
| Funciona em `calc()`? | Não (usa operadores SASS) | Sim |
| Ideal para | Valores de design estáticos | Temas dinâmicos, adaptação responsiva |

::: {.callout-note}
#### Usar os dois em conjunto
Uma boa prática é definir valores em variáveis SASS e expô-los como custom
properties no `:root`, combinando o melhor de ambos:

```scss
$cor-primaria: #5f4fb8;

:root {
  --cor-primaria: #{$cor-primaria};  /* interpolação: #{} converte para string */
}
```

O JavaScript (a linguagem de programação dos browsers, que permite modificar
a página depois de carregada) pode ler e alterar `--cor-primaria` em tempo de
execução; o SASS usa `$cor-primaria` para cálculos durante a compilação.
:::

## Nesting {#sec-07-nesting}

### Hierarquia no CSS {#sec-07-nesting-hierarquia}

O nesting permite escrever regras CSS aninhadas, espelhando a hierarquia do
HTML. O compilador transforma cada nível aninhado num selector descendente:

```scss
/* SCSS */
.card {
  background: white;
  border-radius: $raio;

  .card__titulo {
    font-size: 1.25rem;
    font-weight: bold;
  }

  .card__corpo {
    padding: $espaco-md;
  }
}
```

```css
/* CSS compilado */
.card { background: white; border-radius: 8px; }
.card .card__titulo { font-size: 1.25rem; font-weight: bold; }
.card .card__corpo  { padding: 1rem; }
```

A relação visual entre `.card` e os seus elementos é imediata no SCSS; no CSS
plano equivalente, tem de ser inferida pelo leitor.

### O `&` — referência ao selector pai {#sec-07-nesting-e}

O `&` representa o selector do bloco pai. Permite escrever pseudo-classes,
pseudo-elementos e modificadores no contexto do componente:

```scss
.btn {
```

### A regra dos 3 níveis {#sec-07-nesting-regra}

O nesting em excesso gera selectores CSS excessivamente específicos e difíceis
de sobrepor:

```scss
/* ✗ Nesting excessivo — 5 níveis */
.header {
  .nav {
    .nav__lista {
      .nav__item {
        .nav__link { color: white; }
        /* compila para: .header .nav .nav__lista .nav__item .nav__link */
      }
    }
  }
}
```

A regra prática é **não ultrapassar 3 níveis de nesting**. Se o quarto nível
for necessário, é geralmente sinal de que o componente deve ser refactorado
em blocos independentes.

```scss
/* ✓ Máximo 3 níveis */
.nav {
  &__lista { list-style: none; display: flex; }
  &__item  { margin: 0 0.5rem; }
  &__link  {
    color: white;
    &:hover { color: $cor-primaria; }  /* 3.º nível — aceitável */
  }
}
```

## Partials e Módulos {#sec-07-partials}

### Partials {#sec-07-partials-def}

Um **partial** é um ficheiro SCSS cujo nome começa com `_` (underscore). O
compilador nunca o processa de forma autónoma — só o inclui quando é importado
por outro ficheiro. Esta convenção permite dividir o CSS em ficheiros temáticos
sem gerar um `.css` individual para cada um:

```
scss/
├── _variables.scss   ← tokens de design (cores, tipografia, espaçamentos)
├── _base.scss        ← reset e estilos globais
├── _components.scss  ← botões, cards, mensagens
├── _layout.scss      ← header, footer, grelha
└── main.scss         ← ponto de entrada — importa todos os anteriores
```

Só o `main.scss` (sem underscore) compila para `css/main.css`. Os partials
são peças que o `main.scss` agrega.

### @import vs @use {#sec-07-partials-import-use}

Existem duas formas de incluir um partial: `@import` (método antigo, em
extinção) e `@use` (método moderno, com namespacing):

```scss
/* main.scss — método @import */
@import 'variables';    /* todas as variáveis ficam globais */
@import 'base';
@import 'components';
@import 'layout';
```

```scss
/* _layout.scss — método @use para módulos nativos */
@use 'sass:color';      /* acesso via color.adjust() */
@use 'sass:math';       /* acesso via math.div() */
```

::: {.callout-note}
#### Por que @import está a ser abandonado?
O `@import` torna todas as variáveis, mixins e funções globais — num projecto
grande, torna-se impossível saber de onde vem cada símbolo. O `@use` resolve
isto com **namespacing**: `color.adjust()` indica explicitamente que a função
vem do módulo `color`. O `@import` será removido no Dart Sass 3.0.
:::

### Módulos nativos do Dart Sass {#sec-07-partials-use-nativos}

O Dart Sass inclui módulos nativos que substituem funções globais em extinção:

```scss
@use 'sass:math';   /* math.div(), math.round(), math.floor(), math.ceil() */
@use 'sass:color';  /* color.adjust(), color.mix(), color.scale() */
@use 'sass:string'; /* string.length(), string.to-upper-case() */
@use 'sass:list';   /* list.length(), list.nth() */
```

## Mixins {#sec-07-mixins}

### @mixin — bloco reutilizável {#sec-07-mixins-def}

Um mixin é um bloco de CSS nomeado que pode ser incluído em qualquer selector.
É a resposta do SASS ao problema da repetição de padrões:

```scss
/* Definição */
@mixin flex-center {
  display: flex;
  align-items: center;
  justify-content: center;
}

/* Uso com @include */
.hero            { @include flex-center; min-height: 100vh; }
.modal           { @include flex-center; position: fixed; inset: 0; }
.cartao__imagem  { @include flex-center; height: 160px; }
```

```css
/* CSS compilado */
.hero           { display:flex; align-items:center; justify-content:center; min-height:100vh; }
.modal          { display:flex; align-items:center; justify-content:center; position:fixed; inset:0; }
.cartao__imagem { display:flex; align-items:center; justify-content:center; height:160px; }
```

### Mixins com parâmetros {#sec-07-mixins-params}

Os mixins aceitam parâmetros, tornando-os equivalentes a funções que geram CSS:

```scss
@use 'sass:color';

@mixin btn($cor-fundo: #5f4fb8, $cor-texto: white) {
  background:    $cor-fundo;
  color:         $cor-texto;
  padding:       0.5rem 1.5rem;
  border:        2px solid transparent;
  border-radius: 6px;
  cursor:        pointer;
  font-weight:   600;
  transition:    background 0.2s ease, transform 0.1s ease;

  &:hover  { background: color.adjust($cor-fundo, $lightness: -10%); }
  &:active { transform: translateY(1px); }
}

/* Com valores por defeito */
.btn-primario { @include btn; }

/* Sobrepondo parâmetros */
.btn-sucesso  { @include btn(#198754); }
.btn-perigo   { @include btn(#dc3545); }
.btn-escuro   { @include btn(#212529, white); border-color: #495057; }
```

## Extend e Placeholders {#sec-07-extend}

### @extend {#sec-07-extend-def}

O `@extend` faz com que um selector herde todos os estilos de outro. Em vez
de duplicar propriedades, o compilador agrupa os selectores numa única regra:

```scss
.mensagem {
  padding: 1rem; border-radius: 6px;
  border-left: 4px solid; font-size: 0.9rem; margin-bottom: 1rem;
}

.mensagem-sucesso {
  @extend .mensagem;
  background: #d1e7dd; border-color: #198754; color: #0a3622;
}

.mensagem-erro {
  @extend .mensagem;
  background: #f8d7da; border-color: #dc3545; color: #58151c;
}
```

```css
/* CSS compilado — selectores agrupados */
.mensagem, .mensagem-sucesso, .mensagem-erro {
  padding: 1rem; border-radius: 6px; border-left: 4px solid;
  font-size: 0.9rem; margin-bottom: 1rem;
}
.mensagem-sucesso { background: #d1e7dd; border-color: #198754; color: #0a3622; }
.mensagem-erro    { background: #f8d7da; border-color: #dc3545; color: #58151c; }
```

### Placeholders `%` {#sec-07-extend-placeholder}

Um **placeholder** é um selector especial que começa com `%`. Não gera CSS
por si próprio — só existe para ser estendido. Resolve o problema de
`@extend .mensagem` incluir a classe base no output mesmo que não seja usada
directamente no HTML:

```scss
/* Placeholder — não gera CSS standalone */
%mensagem-base {
  padding: 1rem; border-radius: 6px;
  border-left: 4px solid; font-size: 0.9rem; margin-bottom: 1rem;
  line-height: 1.5;
}

.mensagem-sucesso {
  @extend %mensagem-base;
  background: #d1e7dd; border-color: #198754; color: #0a3622;
}

.mensagem-aviso {
  @extend %mensagem-base;
  background: #fff3cd; border-color: #ffc107; color: #664d03;
}
```

```css
/* CSS compilado — sem .mensagem-base a mais */
.mensagem-sucesso, .mensagem-aviso {
  padding: 1rem; border-radius: 6px; border-left: 4px solid;
  font-size: 0.9rem; margin-bottom: 1rem; line-height: 1.5;

.mensagem-sucesso { background: #d1e7dd; border-color: #198754; color: #0a3622; }
.mensagem-aviso   { background: #fff3cd; border-color: #ffc107; color: #664d03; }
```

### @extend vs @mixin — quando usar cada um {#sec-07-extend-vs-mixin}

| | `@extend` / `%placeholder` | `@mixin` |
|:---|:---:|:---:|
| Aceita parâmetros? | Não | Sim |
| Output CSS | Agrupa selectores | Repete propriedades |
| Pode ser usado em media queries? | Não | Sim |
| Ideal para | Variantes de um base sem parâmetros | Padrões com variações configuráveis |

::: {.callout-note}
#### Regra prática
Usar `%placeholder` + `@extend` quando as variantes partilham exactamente o
mesmo bloco base e não há parâmetros. Usar `@mixin` quando há parâmetros ou
quando o padrão é usado dentro de media queries.
:::

## Funções e Operadores {#sec-07-funcoes}

### Operadores aritméticos {#sec-07-funcoes-aritmetica}

O SASS suporta operações aritméticas directamente nas propriedades CSS:

```scss
$tamanho-base: 16px;
$espaco-base:  1rem;

.titulo    { font-size: $tamanho-base * 2;   }  /* 32px  */
.subtitulo { font-size: $tamanho-base * 1.5; }  /* 24px  */
.espaco-lg { margin:    $espaco-base * 2;    }  /* 2rem  */
```

O operador `/` foi **removido como operador de divisão** no Dart Sass moderno,
pois conflituava com a sintaxe CSS legítima `font: 14px/1.5`. A divisão é
agora feita com `math.div()`:

```scss
@use 'sass:math';

$tamanho-base: 16px;
$espaco-base:  1rem;
$num-colunas:  3;

$tamanho-pequeno: math.div($tamanho-base, 1.25);  /* 12.8px   */
$espaco-sm:       math.div($espaco-base, 2);      /* 0.5rem   */
$largura-coluna:  math.div(100%, $num-colunas);   /* 33.333%  */
```

::: {.callout-note}
#### Por que o `/` foi removido?
Em CSS, `font: 14px/1.5` é sintaxe válida (tamanho/line-height). Para evitar
ambiguidade, o Dart Sass removeu `/` como operador de divisão. O compilador
emite um aviso de extinção se `/` for usado como divisão sem `math.div()`.
:::

### Funções de cor — color.adjust() {#sec-07-funcoes-cor}

O módulo `sass:color` oferece funções para manipular cores de forma
programática, eliminando a necessidade de calcular manualmente variantes mais
claras ou escuras:

```scss
@use 'sass:color';

$cor-primaria: #5f4fb8;

.btn {
  background: $cor-primaria;

  &:hover {
    background: color.adjust($cor-primaria, $lightness: -10%);  /* mais escuro */
  }

  &:disabled {
    background: color.adjust($cor-primaria, $lightness: 30%);   /* desbotado */
  }
}

/* Outros usos comuns */
$cor-sombra: color.adjust($cor-primaria, $alpha: -0.7);         /* 30% opacidade */
$cor-borda:  color.adjust($cor-primaria, $saturation: -20%);    /* menos saturada */
```

### _variables.scss — base de tudo {#sec-07-exemplo-variables}


### _components.scss — mixin e placeholder em acção {#sec-07-exemplo-components}

```scss

/* ── Mixin de botão com parâmetros ───────────── */
@mixin btn($cor-fundo: $cor-primaria, $cor-txt: $cor-branco) {
  background:    $cor-fundo;
  color:         $cor-txt;
  padding:       $espaco-sm $espaco-lg;
  border:        2px solid transparent;
  border-radius: $raio;
  cursor:        pointer;
  font-size:     $tamanho-base;
  font-family:   $fonte-base;
  transition:    background 0.2s ease, transform 0.1s ease;

  &:hover  { background: color.adjust($cor-fundo, $lightness: -10%); }
  &:active {
    background: color.adjust($cor-fundo, $lightness: -18%);
    transform: translateY(1px);
  }
}

.btn-primario { @include btn($cor-primaria); }
.btn-escuro   { @include btn($cor-secundaria); }
.btn-contorno {
  @include btn($cor-branco, $cor-primaria);
  border-color: $cor-primaria;
  &:hover { background: color.adjust($cor-primaria, $lightness: 38%); }
}

/* ── Placeholder para mensagens ──────────────── */
%mensagem-base {
  padding:       $espaco-md;
  border-radius: $raio;
  border-left:   4px solid;
  font-size:     $tamanho-pequeno;
  margin-bottom: $espaco-md;
  line-height:   1.5;
}

.mensagem-sucesso {
  @extend %mensagem-base;
  background:   color.adjust(#28a745, $lightness: 48%);
  border-color: #28a745;
  color:        color.adjust(#28a745, $lightness: -12%);
}

.mensagem-erro {
  @extend %mensagem-base;
  background:   color.adjust(#dc3545, $lightness: 38%);
  border-color: #dc3545;
  color:        color.adjust(#dc3545, $lightness: -12%);
}
```

Neste ficheiro convergem três conceitos do capítulo: o mixin parametrizado
gera três variantes de botão a partir de uma única definição; o placeholder
evita duplicação nas mensagens sem poluir o CSS com selectores desnecessários;
o `color.adjust()` calcula automaticamente as variantes hover e as cores de
estado.

```scss
  gap: $espaco-lg;

  @media (min-width: $bp-sm) { grid-template-columns: repeat(2, 1fr); }
  @media (min-width: $bp-md) { grid-template-columns: repeat($num-colunas, 1fr); }
}

/* Header */
.header {
  background: $cor-secundaria;
  padding: $espaco-md $espaco-lg;

  &__interior {
    max-width: 1100px; margin: 0 auto;
    display: flex; justify-content: space-between; align-items: center;
  }

  &__nav {
    display: flex; gap: $espaco-lg; list-style: none;
    a {
      color: color.adjust($cor-primaria, $lightness: 20%);
      font-size: $tamanho-pequeno;
      transition: color 0.2s ease;
      &:hover { color: $cor-secundaria; }
    }
  }
}

/* Responsividade do header */
@media (max-width: $bp-sm) {
  .header__interior { flex-direction: column; align-items: flex-start; gap: $espaco-sm; }
  .header__nav      { flex-direction: column; gap: $espaco-sm; }
}
```

As media queries estão aninhadas dentro de `.grelha`, tirando partido do
nesting SASS — a relação entre o componente e o seu comportamento responsivo
é visualmente imediata, sem necessitar de um bloco `@media` separado no final
do ficheiro.

## Síntese do Capítulo {#sec-07-sintese}

O **SASS** resolve três problemas estruturais do CSS: valores repetidos, falta
de hierarquia visível e manutenção difícil. É um pré-processador — compila
para CSS antes de chegar ao browser. O browser nunca vê código SASS.

As **variáveis** (`$nome: valor`) centralizam tokens de design. Alterar uma
variável propaga a mudança para todo o projecto. Distinguem-se das CSS Custom
Properties: as variáveis SASS desaparecem na compilação; as `--var` persistem
no browser e podem ser alteradas em tempo de execução por JavaScript.

O **nesting** espelha a hierarquia do HTML no CSS. O `&` referencia o selector
pai, permitindo escrever pseudo-classes, pseudo-elementos e modificadores BEM
no contexto do componente. A regra dos 3 níveis evita selectores excessivamente
específicos.

Os **partials** (`_ficheiro.scss`) dividem o CSS em módulos temáticos. Só o
ponto de entrada (sem underscore) compila para `.css`. O `@use` é o método
moderno de inclusão — com namespacing, substitui o `@import` em extinção.

Os **mixins** (`@mixin` / `@include`) encapsulam padrões CSS reutilizáveis
com parâmetros. O **extend** e os **placeholders** (`%`) reutilizam blocos
base sem parâmetros, agrupando selectores no CSS compilado.

As **funções** dos módulos `sass:math` e `sass:color` permitem cálculos e
manipulações de cor em tempo de compilação. `math.div()` substitui o operador
`/`, removido para evitar conflito com a sintaxe CSS de `font`.

## Exercícios {#sec-07-exercicios}

::: {.callout-caution}
#### Exercício 7.1 — Variáveis e estrutura base
Crie um projecto SASS com a estrutura mínima: `_variables.scss`, `_base.scss`,
`main.scss`.

a) Em `_variables.scss`, defina: cor primária, cor de texto, tamanho base (16px),
   espaçamento base (1rem) e raio de bordo (8px).
b) Calcule com `math.div()`: tamanho pequeno (base ÷ 1.25) e espaçamento
   pequeno (base ÷ 2). Confirme os valores no CSS compilado.
c) Em `_base.scss`, aplique as variáveis ao `body` (font-size, color, font-family).
d) Em `main.scss`, importe os dois partials na ordem correcta.

Explique por que a ordem de importação dos partials importa.
:::

::: {.callout-caution}
#### Exercício 7.2 — Nesting e o `&`
Crie um componente `.cartao` com nesting SASS:

a) `.cartao` com fundo branco, raio de bordo e sombra.
b) `.cartao__titulo` e `.cartao__corpo` como elementos aninhados.
c) `.cartao:hover` com `transform: translateY(-4px)` e sombra mais intensa
   — usando `&`.
d) Não ultrapasse 3 níveis de nesting.

Escreva o CSS compilado equivalente à mão e compare com o output real do
compilador.
:::

::: {.callout-caution}
#### Exercício 7.3 — Mixin de botão
Crie um mixin `btn` com dois parâmetros: `$cor-fundo` (por defeito: cor
primária) e `$cor-texto` (por defeito: branco).

a) O mixin deve incluir: padding, border-radius, cursor e transição de background.
b) O estado `&:hover` deve escurecer `$cor-fundo` com
   `color.adjust(..., $lightness: -10%)`.
c) Crie três variantes: `.btn-primario`, `.btn-sucesso` (#198754) e
   `.btn-perigo` (#dc3545).
d) Adicione `.btn-contorno`: fundo transparente, texto e borda na cor primária.
:::

::: {.callout-caution}
#### Exercício 7.4 — Extend e placeholders
Crie um sistema de mensagens com três variantes (sucesso, aviso, erro):

a) Defina um placeholder `%alerta-base` com o estilo comum (padding,
   border-left, border-radius, font-size).
b) Crie `.alerta-sucesso`, `.alerta-aviso` e `.alerta-erro` usando
   `@extend %alerta-base`.
c) Cada variante deve ter cor de fundo, borda e texto distintas.
d) Examine o CSS compilado e explique por que o placeholder não aparece
   como selector no output.

Compare o output com o que seria gerado usando `@mixin` em vez de `@extend`.
:::

::: {.callout-caution}
#### Exercício 7.5 — Projecto completo com partials
Organize um mini-projecto com a estrutura completa de quatro partials:

a) `_variables.scss` — cores, tipografia, espaçamentos, breakpoints
   `$bp-sm: 576px` e `$bp-md: 768px`.
b) `_layout.scss` — `.grelha` com `@use 'sass:math'`,
   `$largura-coluna: math.div(100%, 3)`, e media queries nested para
   1 → 2 → 3 colunas.
c) `_components.scss` — pelo menos um mixin e um placeholder, cada um
   reutilizado em dois selectores diferentes.
d) `main.scss` — importa os três partials.

Redimensione o browser para verificar o comportamento responsivo da grelha
nos três breakpoints.
:::
