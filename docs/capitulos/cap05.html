5  Layout CSS — Box Model, Flexbox e Grid

Ferramentas e Aplicações Multimédia

Autor

Data de Publicação

Introdução

O layout de uma página web é o processo pelo qual os elementos HTML são posicionados no espaço visual disponível. Durante décadas, este processo foi dominado por técnicas improvisadas — tabelas HTML, floats — que nunca foram concebidas para o propósito. O CSS moderno oferece hoje dois sistemas de layout dedicados: o Flexbox e o CSS Grid, cada um optimizado para um conjunto específico de problemas.

Neste capítulo estuda-se o box model — o modelo fundamental que governa o espaço ocupado por qualquer elemento HTML — a grelha manual de 12 colunas construída com floats, e os dois sistemas modernos de layout: Flexbox para distribuição unidimensional e Grid para composição bidimensional.

5.1 O Box Model

5.1.1 Definição e camadas

Cada elemento HTML é representado pelo browser como um rectângulo com quatro camadas concêntricas — o box model. Compreender estas camadas é condição prévia para qualquer trabalho de layout em CSS.

As quatro camadas, da mais interna para a mais externa, são:

  • Content — a área onde o conteúdo real reside: texto, imagem, vídeo. As propriedades width e height definem as dimensões desta camada (no modo content-box, o padrão histórico do CSS).
  • Padding — espaço interior entre o conteúdo e a fronteira. Tem a cor de fundo do elemento e aumenta a área clicável sem afectar a posição do conteúdo.
  • Border — contorno do elemento. Tem espessura (border-width), estilo (border-style) e cor (border-color) configuráveis.
  • Margin — espaço exterior entre o elemento e os seus vizinhos. As margens são transparentes — não herdam a cor de fundo do elemento.

5.1.2 box-sizing: content-box vs border-box

O comportamento histórico do CSS — definido na propriedade box-sizing: content-box — determina que width e height se aplicam apenas à área de conteúdo. O padding e a border são adicionados por fora, aumentando as dimensões totais do elemento. Este comportamento é sistematicamente contra-intuitivo:

/* content-box: largura total = 200 + 20 + 20 + 2 + 2 = 244px */
.caixa {
  width: 200px;
  padding: 20px;
  border: 2px solid #333;
  box-sizing: content-box; /* comportamento por omissão */
}

O modo border-box inclui o padding e a border dentro das dimensões declaradas — o que corresponde à intuição natural de “esta caixa tem 200px de largura”:

/* border-box: largura total = 200px (exacamente) */
.caixa {
  width: 200px;
  padding: 20px;
  border: 2px solid #333;
  box-sizing: border-box; /* recomendado */
}

A prática moderna é aplicar border-box globalmente com o selector universal:

*, *::before, *::after {
  box-sizing: border-box;
}

Esta regra está presente em todos os frameworks CSS modernos, incluindo o Bootstrap.

Por que o content-box existe?

O content-box é o comportamento original do CSS 1 (1996), adoptado por razões de compatibilidade com os browsers da época. O Internet Explorer 6 e anteriores implementavam border-box por omissão, criando uma incompatibilidade histórica. O CSS 3 resolveu o problema tornando ambos os modos explicitamente configuráveis.

5.1.3 Margin, padding e border em CSS

As quatro camadas do box model são controladas por propriedades CSS com sintaxe uniforme. Todas suportam a notação shorthand — um único valor aplica-se a todos os lados; dois valores aplicam-se a {cima/baixo, esquerda/direita}; quatro valores aplicam-se a {cima, direita, baixo, esquerda} (sentido horário).

/* margin — shorthand */
.el { margin: 16px; }              /* todos os lados */
.el { margin: 8px 16px; }          /* cima/baixo | esquerda/direita */
.el { margin: 4px 8px 12px 6px; }  /* cima | direita | baixo | esquerda */

/* padding — mesma lógica */
.el { padding: 24px; }
.el { padding: 12px 24px; }

/* border */
.el { border: 1px solid #333; }
.el { border-top: 2px dashed #c00; border-bottom: none; }

/* centrar horizontalmente com margin auto */
.contentor {
  width: 960px;
  margin: 0 auto; /* margens laterais automáticas = centrado */
}

5.1.4 Colapso de margens

As margens verticais entre elementos de bloco adjacentes colapsam: em vez de se somarem, prevalece a maior das duas. Este comportamento designa-se margin collapsing e é uma fonte frequente de confusão.

/* Dois parágrafos consecutivos: margin-bottom 24px + margin-top 16px */
/* Espaço real entre eles: max(24, 16) = 24px — não 40px */
p { margin-bottom: 24px; }
h2 { margin-top: 16px; }

O colapso de margens não ocorre em contextos flex nem em contextos grid — apenas entre blocos no fluxo normal do documento.

5.2 Float e Grelha Manual

5.2.1 O float e o seu propósito original

A propriedade float foi concebida para um propósito editorial específico: permitir que texto flua em torno de uma imagem, à semelhança do comportamento de revistas impressas. Os valores possíveis são left, right e none.

/* Imagem com texto a fluir à direita */
img.destaque {
  float: left;
  margin: 0 16px 8px 0;
}

Durante os anos 2000, na ausência de alternativas nativas de layout, os programadores adaptaram o float para construir layouts multi-coluna — um uso para o qual nunca foi desenhado, com consequências que veremos a seguir.

5.2.2 O colapso do contentor e o clearfix

Um elemento flutuante sai do fluxo normal do documento. O resultado prático é que o seu contentor pai não contabiliza a sua altura — o contentor colapsa como se o elemento flutuante não existisse:

/* Problema: .pai não "vê" .filho flutuante */
.pai   { background: #eee; } /* altura = 0, mesmo com filho */
.filho { float: left; width: 200px; height: 150px; }

A solução clássica é o clearfix — uma técnica que força o contentor a englobar os seus filhos flutuantes através de um pseudo-elemento vazio:

/* Clearfix moderno com ::after */
.clearfix::after {
  content: "";
  display: table;
  clear: both;
}

Aplicando .clearfix ao contentor, este passa a incluir a altura dos seus filhos flutuantes.

5.2.3 Grelha de 12 colunas com floats

O sistema de 12 colunas é o layout mais utilizado no design web. A escolha de 12 justifica-se pela sua divisibilidade: admite colunas de 1, 2, 3, 4, 6 e 12 partes iguais, cobrindo a maioria dos layouts editoriais.

Numa implementação manual com floats, cada coluna é uma percentagem da largura do contentor. Com um gutter (espaço entre colunas) de 2%, cada coluna tem largura \((100\% - 11 \times 2\%) / 12 \approx 6{,}5\%\). Na prática simplifica-se omitindo os gutters e usando padding interno:

/* Contentor */
.row { width: 100%; }
.row::after { content: ""; display: table; clear: both; }

/* Colunas — percentagens de 12 */
[class^="col-"] { float: left; }
.col-1  { width:  8.333%; }
.col-2  { width: 16.666%; }
.col-3  { width: 25%; }
.col-4  { width: 33.333%; }
.col-6  { width: 50%; }
.col-8  { width: 66.666%; }
.col-9  { width: 75%; }
.col-12 { width: 100%; }

Uso em HTML:

<div class="row">
  <div class="col-8">Conteúdo principal</div>
  <div class="col-4">Sidebar</div>
</div>

As colunas de uma linha devem sempre somar 12. Se a soma for inferior a 12, sobra espaço; se for superior, a última coluna salta para a linha seguinte.

5.2.4 Limitações do float como sistema de layout

Apesar da sua ubiquidade durante mais de uma década, o float apresenta limitações estruturais quando usado para layout:

  • Saída do fluxo — os elementos flutuantes não participam no cálculo de altura do contentor, exigindo clearfix em cada linha.
  • Alinhamento vertical impossível — centrar verticalmente dois elementos de alturas diferentes é impraticável com floats.
  • Reordenação visual — a ordem visual é determinada pela ordem no HTML; reordenar visualmente exige alterar o markup.
  • Responsividade complexa — adaptar uma grelha float a múltiplos breakpoints exige muitas media queries.

Estas limitações motivaram o desenvolvimento do Flexbox (2009–2012) e do CSS Grid (2017), que resolvem os problemas de layout de forma nativa.

5.3 Flexbox

5.3.1 O modelo flex

O Flexbox (Flexible Box Layout) é um modelo de layout unidimensional — distribui elementos ao longo de uma direcção: linha ou coluna. É optimizado para:

  • Distribuir espaço entre elementos ao longo de um eixo
  • Alinhar elementos relativamente a um eixo
  • Adaptar automaticamente o tamanho dos elementos ao espaço disponível

O modelo assenta na distinção entre contentor flex (flex container) e filhos flex (flex items):

/* Activar flex no contentor */
.contentor { display: flex; }

/* Todos os filhos directos tornam-se automaticamente flex items */
Unidimensional vs bidimensional

O Flexbox é unidimensional: trabalha numa direcção de cada vez (linha ou coluna). O CSS Grid é bidimensional: trabalha em linha e coluna simultaneamente. Esta diferença determina qual dos dois usar em cada situação.

5.3.2 Eixo principal e eixo transversal

O Flexbox organiza-se em torno de dois eixos perpendiculares:

  • Eixo principal (main axis) — a direcção em que os filhos são distribuídos. Definido por flex-direction.
  • Eixo transversal (cross axis) — perpendicular ao eixo principal.
.contentor {
  display: flex;
  flex-direction: row;    /* padrão: esquerda → direita */
  /* flex-direction: row-reverse; — direita → esquerda */
  /* flex-direction: column;      — cima → baixo */
  /* flex-direction: column-reverse; — baixo → cima */
}

Quando flex-direction: row, o eixo principal é horizontal e o transversal é vertical. Quando flex-direction: column, invertem-se os papéis.

5.3.3 Propriedades do contentor flex

As propriedades que controlam o comportamento do contentor afectam todos os filhos em conjunto:

.contentor {
  display: flex;

  /* Direcção e quebra */
  flex-direction: row;           /* row | row-reverse | column | column-reverse */
  flex-wrap: nowrap;             /* nowrap | wrap | wrap-reverse */
  flex-flow: row nowrap;         /* shorthand: direction + wrap */

  /* Distribuição no eixo principal */
  justify-content: flex-start;  /* flex-start | flex-end | center |
                                    space-between | space-around | space-evenly */

  /* Alinhamento no eixo transversal */
  align-items: stretch;         /* stretch | flex-start | flex-end |
                                    center | baseline */

  /* Alinhamento de múltiplas linhas */
  align-content: flex-start;    /* (só quando flex-wrap: wrap) */

  /* Espaço entre filhos */
  gap: 16px;                    /* equivalente a row-gap + column-gap */
}

5.3.4 Propriedades dos filhos flex

Cada filho pode ter propriedades individuais que sobrepõem ou complementam o comportamento definido pelo contentor:

.filho {
  /* Crescimento, encolhimento e tamanho base */
  flex-grow:   0;    /* factor de crescimento (0 = não cresce) */
  flex-shrink: 1;    /* factor de encolhimento (1 = encolhe proporcionalmente) */
  flex-basis:  auto; /* tamanho base antes da distribuição de espaço */
  flex: 1;           /* shorthand: grow=1, shrink=1, basis=0 */

  /* Alinhamento individual (sobrepõe align-items do contentor) */
  align-self: center; /* auto | flex-start | flex-end | center | stretch */

  /* Ordem visual (sem alterar o HTML) */
  order: 0;          /* valor inteiro; menor = aparece primeiro */
}

A propriedade flex: 1 é a mais usada na prática: aplica flex-grow: 1, flex-shrink: 1 e flex-basis: 0, o que faz o filho ocupar uma quota igual do espaço disponível.

5.3.5 Casos de uso típicos do Flexbox

/* Centrar vertical e horizontalmente */
.contentor {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* Navbar com logo à esquerda e links à direita */
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

/* Linha de cards com alturas iguais */
.cards {
  display: flex;
  gap: 16px;
}
.card {
  flex: 1;           /* distribuição igual */
  display: flex;
  flex-direction: column;
}
.card-footer {
  margin-top: auto;  /* empurra o rodapé para o fundo do card */
}

/* Layout principal com sidebar */
.layout {
  display: flex;
  gap: 24px;
}
.conteudo { flex: 3; }
.sidebar  { flex: 1; }

5.4 CSS Grid

5.4.1 O modelo de grelha bidimensional

O CSS Grid é um sistema de layout bidimensional — posiciona elementos ao longo de dois eixos simultâneos: linhas e colunas. É optimizado para:

  • Layouts de página completos (header, nav, main, sidebar, footer)
  • Grelhas de conteúdo regulares (galeria de imagens, lista de produtos)
  • Posicionamento preciso de elementos em relação a uma grelha
/* Activar grid no contentor */
.contentor { display: grid; }

O conceito central é a grelha explícita — o programador declara quantas colunas e linhas existem, e o browser coloca os filhos segundo essas regras.

5.4.2 Definir colunas e linhas

As colunas e linhas são definidas pelas propriedades grid-template-columns e grid-template-rows:

.grelha {
  display: grid;

  /* 3 colunas: 200px fixo, 1fr flexível, 1fr flexível */
  grid-template-columns: 200px 1fr 1fr;

  /* 2 linhas automáticas */
  grid-template-rows: auto;

  /* Espaço entre células */
  gap: 16px;
}

/* A unidade fr (fraction) distribui o espaço disponível proporcionalmente */
/* 1fr + 2fr + 1fr = 4 partes: 25% + 50% + 25% */
.grelha-flex {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr;
  gap: 12px;
}

A função repeat() evita repetição para grelhas regulares:

/* 12 colunas iguais */
.grelha-12 {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  gap: 16px;
}

/* Colunas responsivas automáticas: mínimo 200px, máximo 1fr */
.grelha-auto {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  gap: 16px;
}

5.4.3 Posicionamento de elementos

Por omissão, o CSS Grid coloca os filhos sequencialmente, célula a célula. Para controlo preciso, podem-se usar as propriedades de posicionamento:

/* Notação: linha-início / linha-fim */
.item-a {
  grid-column: 1 / 3;   /* ocupa colunas 1 e 2 */
  grid-row: 1 / 2;      /* ocupa a linha 1 */
}

/* Alternativa com span: ocupa N colunas a partir da posição actual */
.item-b {
  grid-column: span 2;  /* ocupa 2 colunas */
}

/* Posicionamento absoluto na grelha */
.destaque {
  grid-column: 2 / 4;
  grid-row: 1 / 3;
}

5.4.4 Template areas

A propriedade grid-template-areas permite nomear zonas da grelha com texto, tornando o layout visualmente declarativo:

.pagina {
  display: grid;
  grid-template-columns: 220px 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header  header"
    "sidebar main"
    "footer  footer";
  min-height: 100vh;
  gap: 0;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

O mapa textual de grid-template-areas reproduz visualmente o layout pretendido. Um ponto (.) representa uma célula vazia.

5.4.5 Flexbox vs CSS Grid — quando usar cada um

A escolha entre Flexbox e Grid não é de exclusão mútua — os dois sistemas coexistem e complementam-se numa mesma página:

Flexbox CSS Grid
Dimensionalidade Unidimensional (1 eixo) Bidimensional (2 eixos)
Controlo A partir do filho (tamanho flexível) A partir do contentor (grelha explícita)
Caso de uso ideal Distribuição de items em linha/coluna Layouts de página, grelhas regulares
Alinhamento Excelente num eixo Excelente nos dois eixos
Reordenação order por item Posicionamento total
Suporte browsers Excelente (IE 11+) Excelente (Chrome 57+, 2017)

Regra prática:

  • Usar Flexbox para componentes — navbar, cards em linha, botões alinhados, centrar um elemento.
  • Usar Grid para a estrutura da página — header/main/sidebar/footer, galerias de imagens, layouts complexos.
/* Exemplo realista: Grid para a página, Flex para os componentes */

/* Estrutura da página — Grid */
.pagina {
  display: grid;
  grid-template-areas:
    "header"
    "main"
    "footer";
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}

/* Navbar dentro do header — Flex */
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 0 24px;
}

/* Grelha de cards dentro do main — Grid */
.galeria {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: 24px;
  padding: 24px;
}

/* Conteúdo interno de cada card — Flex */
.card {
  display: flex;
  flex-direction: column;
}
.card-footer { margin-top: auto; }

5.5 Síntese do Capítulo

O box model é o fundamento de todo o layout CSS: cada elemento ocupa um rectângulo com quatro camadas — conteúdo, padding, border e margin. A propriedade box-sizing: border-box torna o comportamento intuitivo e é adoptada por todos os frameworks modernos.

O float foi o mecanismo de layout dominante durante a primeira décade da web responsiva. A grelha manual de 12 colunas, construída com floats e percentagens, demonstra os princípios que o Bootstrap depois automatizou. As suas limitações — colapso do contentor, impossibilidade de alinhamento vertical, complexidade responsiva — motivaram os sistemas modernos.

O Flexbox resolve o layout unidimensional: distribui elementos ao longo de uma linha ou coluna, alinha-os nos dois eixos e adapta os seus tamanhos ao espaço disponível. É o sistema adequado para componentes — navbars, filas de cards, centração de elementos.

O CSS Grid resolve o layout bidimensional: posiciona elementos em relação a uma grelha de linhas e colunas definida explicitamente no contentor. É o sistema adequado para a estrutura da página e para grelhas regulares. Os dois sistemas não são concorrentes — coexistem e complementam-se num mesmo projecto.

5.6 Exercícios

Exercício 5.1 — Box model e box-sizing

Dado o seguinte CSS em modo content-box:

.caixa {
  width: 300px;
  padding: 20px;
  border: 3px solid #333;
  margin: 16px;
}
  1. Qual é a largura total ocupada pelo elemento (incluindo border)?
  2. Qual seria a largura total em modo border-box?
  3. Escreva a regra global que activa border-box em todos os elementos.

Verificação: a) 346px; b) 300px; c) *, *::before, *::after { box-sizing: border-box; }

Exercício 5.2 — Grelha de 12 colunas com floats

Construa, apenas com CSS e a técnica de floats, um layout com:

  • Uma linha com três colunas de igual largura (4 + 4 + 4 colunas)
  • Uma linha com uma coluna de conteúdo principal e uma sidebar (8 + 4 colunas)

Inclua o clearfix necessário para que o contentor não colapse.

Exercício 5.3 — Flexbox

Usando apenas Flexbox (sem Grid, sem floats), implemente:

  1. Um contentor que centra um único elemento filho horizontal e verticalmente.
  2. Uma barra de navegação com o logótipo à esquerda e quatro links à direita, todos alinhados verticalmente ao centro.
  3. Uma linha de quatro cards que ocupa toda a largura disponível, com espaço igual entre os cards mas sem espaço nas extremidades.
Exercício 5.4 — CSS Grid

Usando CSS Grid e grid-template-areas, implemente o seguinte layout de página:

+-----------------------+
|        header         |
+-----------+-----------+
|  sidebar  |   main    |
+-----------+-----------+
|        footer         |
+-----------------------+

Condições: header e footer com altura fixa de 60px; sidebar com largura fixa de 240px; main ocupa o espaço restante; a página ocupa sempre 100% da altura do viewport.

Exercício 5.5 — Escolha do sistema de layout

Para cada situação, indique se usaria Flexbox, Grid ou ambos, e justifique:

  1. Distribuir cinco ícones de redes sociais em linha, com espaçamento igual.
  2. Construir o layout completo de uma página com header, dois sidebars e área central.
  3. Criar uma galeria fotográfica responsiva com colunas que se adaptam automaticamente.
  4. Alinhar verticalmente o texto e o ícone dentro de um botão.

## Introdução {#sec-05-intro .unnumbered}

O layout de uma página web é o processo pelo qual os elementos HTML são
posicionados no espaço visual disponível. Durante décadas, este processo foi
dominado por técnicas improvisadas — tabelas HTML, floats — que nunca foram
concebidas para o propósito. O CSS moderno oferece hoje dois sistemas de layout
dedicados: o **Flexbox** e o **CSS Grid**, cada um optimizado para um conjunto
específico de problemas.

Neste capítulo estuda-se o box model — o modelo fundamental que governa o
espaço ocupado por qualquer elemento HTML — a grelha manual de 12 colunas
construída com floats, e os dois sistemas modernos de layout: Flexbox para
distribuição unidimensional e Grid para composição bidimensional.

## O Box Model {#sec-05-box-model}

### Definição e camadas {#sec-05-box-model-def}

Cada elemento HTML é representado pelo browser como um rectângulo com quatro
camadas concêntricas — o **box model**. Compreender estas camadas é condição
prévia para qualquer trabalho de layout em CSS.

As quatro camadas, da mais interna para a mais externa, são:

- **Content** — a área onde o conteúdo real reside: texto, imagem, vídeo.
  As propriedades `width` e `height` definem as dimensões desta camada
  (no modo `content-box`, o padrão histórico do CSS).
- **Padding** — espaço interior entre o conteúdo e a fronteira. Tem a cor
  de fundo do elemento e aumenta a área clicável sem afectar a posição do
  conteúdo.
- **Border** — contorno do elemento. Tem espessura (`border-width`), estilo

- **Margin** — espaço exterior entre o elemento e os seus vizinhos.
  As margens são transparentes — não herdam a cor de fundo do elemento.

### box-sizing: content-box vs border-box {#sec-05-box-sizing}

O comportamento histórico do CSS — definido na propriedade `box-sizing:
content-box` — determina que `width` e `height` se aplicam **apenas à área
de conteúdo**. O padding e a border são adicionados por fora, aumentando as
dimensões totais do elemento. Este comportamento é sistematicamente
contra-intuitivo:

```css
/* content-box: largura total = 200 + 20 + 20 + 2 + 2 = 244px */
.caixa {
  width: 200px;
  padding: 20px;
  border: 2px solid #333;
  box-sizing: content-box; /* comportamento por omissão */
}
```

O modo `border-box` inclui o padding e a border dentro das dimensões
declaradas — o que corresponde à intuição natural de "esta caixa tem 200px
de largura":

```css
/* border-box: largura total = 200px (exacamente) */
.caixa {
  width: 200px;
  padding: 20px;
  border: 2px solid #333;
  box-sizing: border-box; /* recomendado */
}
```

A prática moderna é aplicar `border-box` globalmente com o selector universal:

```css
*, *::before, *::after {
  box-sizing: border-box;
}
```

Esta regra está presente em todos os frameworks CSS modernos, incluindo
o Bootstrap.

::: {.callout-note}
#### Por que o content-box existe?
O `content-box` é o comportamento original do CSS 1 (1996), adoptado por
razões de compatibilidade com os browsers da época. O Internet Explorer 6
e anteriores implementavam `border-box` por omissão, criando uma
incompatibilidade histórica. O CSS 3 resolveu o problema tornando ambos
os modos explicitamente configuráveis.
:::

### Margin, padding e border em CSS {#sec-05-propriedades-espaciais}

As quatro camadas do box model são controladas por propriedades CSS com
sintaxe uniforme. Todas suportam a notação shorthand — um único valor aplica-se
a todos os lados; dois valores aplicam-se a {cima/baixo, esquerda/direita};
quatro valores aplicam-se a {cima, direita, baixo, esquerda} (sentido horário).

```css
/* margin — shorthand */
.el { margin: 16px; }              /* todos os lados */
.el { margin: 8px 16px; }          /* cima/baixo | esquerda/direita */
.el { margin: 4px 8px 12px 6px; }  /* cima | direita | baixo | esquerda */

/* padding — mesma lógica */
.el { padding: 24px; }
.el { padding: 12px 24px; }

/* border */
.el { border: 1px solid #333; }
.el { border-top: 2px dashed #c00; border-bottom: none; }

/* centrar horizontalmente com margin auto */
.contentor {
  width: 960px;
  margin: 0 auto; /* margens laterais automáticas = centrado */
}
```

### Colapso de margens {#sec-05-margin-collapse}

As margens verticais entre elementos de bloco adjacentes **colapsam**: em vez
de se somarem, prevalece a maior das duas. Este comportamento designa-se
*margin collapsing* e é uma fonte frequente de confusão.

```css
/* Dois parágrafos consecutivos: margin-bottom 24px + margin-top 16px */
/* Espaço real entre eles: max(24, 16) = 24px — não 40px */
p { margin-bottom: 24px; }
h2 { margin-top: 16px; }
```

O colapso de margens não ocorre em contextos flex nem em contextos grid —
apenas entre blocos no fluxo normal do documento.

## Float e Grelha Manual {#sec-05-float}

### O float e o seu propósito original {#sec-05-float-origem}

A propriedade `float` foi concebida para um propósito editorial específico:
permitir que texto flua em torno de uma imagem, à semelhança do comportamento
de revistas impressas. Os valores possíveis são `left`, `right` e `none`.

```css
/* Imagem com texto a fluir à direita */
img.destaque {
  float: left;
  margin: 0 16px 8px 0;
}
```

Durante os anos 2000, na ausência de alternativas nativas de layout, os
programadores adaptaram o float para construir layouts multi-coluna — um
uso para o qual nunca foi desenhado, com consequências que veremos a seguir.

### O colapso do contentor e o clearfix {#sec-05-clearfix}

Um elemento flutuante **sai do fluxo normal** do documento. O resultado
prático é que o seu contentor pai não contabiliza a sua altura — o contentor
colapsa como se o elemento flutuante não existisse:

```css
/* Problema: .pai não "vê" .filho flutuante */
.pai   { background: #eee; } /* altura = 0, mesmo com filho */
.filho { float: left; width: 200px; height: 150px; }
```

A solução clássica é o **clearfix** — uma técnica que força o contentor a
englobar os seus filhos flutuantes através de um pseudo-elemento vazio:

```css
/* Clearfix moderno com ::after */
.clearfix::after {
  content: "";
  display: table;
  clear: both;
}
```

Aplicando `.clearfix` ao contentor, este passa a incluir a altura dos seus
filhos flutuantes.

### Grelha de 12 colunas com floats {#sec-05-grelha-12}

O sistema de 12 colunas é o layout mais utilizado no design web. A escolha
de 12 justifica-se pela sua divisibilidade: admite colunas de 1, 2, 3, 4, 6
e 12 partes iguais, cobrindo a maioria dos layouts editoriais.

Numa implementação manual com floats, cada coluna é uma percentagem da
largura do contentor. Com um *gutter* (espaço entre colunas) de 2%, cada
coluna tem largura $(100\% - 11 \times 2\%) / 12 \approx 6{,}5\%$. Na prática
simplifica-se omitindo os gutters e usando padding interno:

```css
/* Contentor */
.row { width: 100%; }
.row::after { content: ""; display: table; clear: both; }

/* Colunas — percentagens de 12 */
[class^="col-"] { float: left; }
.col-1  { width:  8.333%; }
.col-2  { width: 16.666%; }
.col-3  { width: 25%; }
.col-4  { width: 33.333%; }
.col-6  { width: 50%; }
.col-8  { width: 66.666%; }
.col-9  { width: 75%; }
.col-12 { width: 100%; }
```

**Uso em HTML:**

```html
<div class="row">
  <div class="col-8">Conteúdo principal</div>
  <div class="col-4">Sidebar</div>
</div>
```

As colunas de uma linha devem sempre somar 12. Se a soma for inferior a 12,
sobra espaço; se for superior, a última coluna salta para a linha seguinte.

### Limitações do float como sistema de layout {#sec-05-float-limitacoes}

Apesar da sua ubiquidade durante mais de uma década, o float apresenta
limitações estruturais quando usado para layout:

- **Saída do fluxo** — os elementos flutuantes não participam no cálculo
  de altura do contentor, exigindo clearfix em cada linha.
- **Alinhamento vertical impossível** — centrar verticalmente dois elementos
  de alturas diferentes é impraticável com floats.
- **Reordenação visual** — a ordem visual é determinada pela ordem no HTML;
  reordenar visualmente exige alterar o markup.
- **Responsividade complexa** — adaptar uma grelha float a múltiplos
  breakpoints exige muitas media queries.

Estas limitações motivaram o desenvolvimento do Flexbox (2009–2012) e do
CSS Grid (2017), que resolvem os problemas de layout de forma nativa.

## Flexbox {#sec-05-flexbox}

### O modelo flex {#sec-05-flex-modelo}

O **Flexbox** (*Flexible Box Layout*) é um modelo de layout unidimensional
— distribui elementos ao longo de **uma** direcção: linha ou coluna. É
optimizado para:

- Distribuir espaço entre elementos ao longo de um eixo
- Alinhar elementos relativamente a um eixo
- Adaptar automaticamente o tamanho dos elementos ao espaço disponível

O modelo assenta na distinção entre **contentor flex** (*flex container*)
e **filhos flex** (*flex items*):

```css
/* Activar flex no contentor */
.contentor { display: flex; }

/* Todos os filhos directos tornam-se automaticamente flex items */
```

::: {.callout-note}
#### Unidimensional vs bidimensional
O Flexbox é **unidimensional**: trabalha numa direcção de cada vez (linha
*ou* coluna). O CSS Grid é **bidimensional**: trabalha em linha *e* coluna
simultaneamente. Esta diferença determina qual dos dois usar em cada situação.
:::

### Eixo principal e eixo transversal {#sec-05-flex-eixos}

O Flexbox organiza-se em torno de dois eixos perpendiculares:

- **Eixo principal** (*main axis*) — a direcção em que os filhos são
  distribuídos. Definido por `flex-direction`.
- **Eixo transversal** (*cross axis*) — perpendicular ao eixo principal.

```css
.contentor {
  display: flex;
  flex-direction: row;    /* padrão: esquerda → direita */
  /* flex-direction: row-reverse; — direita → esquerda */
  /* flex-direction: column;      — cima → baixo */
  /* flex-direction: column-reverse; — baixo → cima */
}
```

Quando `flex-direction: row`, o eixo principal é horizontal e o transversal
é vertical. Quando `flex-direction: column`, invertem-se os papéis.

### Propriedades do contentor flex {#sec-05-flex-contentor}

As propriedades que controlam o comportamento do contentor afectam todos
os filhos em conjunto:

```css
.contentor {
  display: flex;

  /* Direcção e quebra */
  flex-direction: row;           /* row | row-reverse | column | column-reverse */
  flex-wrap: nowrap;             /* nowrap | wrap | wrap-reverse */
  flex-flow: row nowrap;         /* shorthand: direction + wrap */

  /* Distribuição no eixo principal */
  justify-content: flex-start;  /* flex-start | flex-end | center |
                                    space-between | space-around | space-evenly */

  /* Alinhamento no eixo transversal */
  align-items: stretch;         /* stretch | flex-start | flex-end |
                                    center | baseline */

  /* Alinhamento de múltiplas linhas */
  align-content: flex-start;    /* (só quando flex-wrap: wrap) */

  /* Espaço entre filhos */
  gap: 16px;                    /* equivalente a row-gap + column-gap */
}
```

### Propriedades dos filhos flex {#sec-05-flex-filhos}

Cada filho pode ter propriedades individuais que sobrepõem ou complementam
o comportamento definido pelo contentor:

```css
.filho {
  /* Crescimento, encolhimento e tamanho base */
  flex-grow:   0;    /* factor de crescimento (0 = não cresce) */
  flex-shrink: 1;    /* factor de encolhimento (1 = encolhe proporcionalmente) */
  flex-basis:  auto; /* tamanho base antes da distribuição de espaço */
  flex: 1;           /* shorthand: grow=1, shrink=1, basis=0 */

  /* Alinhamento individual (sobrepõe align-items do contentor) */
  align-self: center; /* auto | flex-start | flex-end | center | stretch */

  /* Ordem visual (sem alterar o HTML) */
  order: 0;          /* valor inteiro; menor = aparece primeiro */
}
```

A propriedade `flex: 1` é a mais usada na prática: aplica `flex-grow: 1`,
`flex-shrink: 1` e `flex-basis: 0`, o que faz o filho ocupar uma quota
igual do espaço disponível.

### Casos de uso típicos do Flexbox {#sec-05-flex-usos}

```css
/* Centrar vertical e horizontalmente */
.contentor {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* Navbar com logo à esquerda e links à direita */
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

/* Linha de cards com alturas iguais */
.cards {
  display: flex;
  gap: 16px;
}
.card {
  flex: 1;           /* distribuição igual */
  display: flex;
  flex-direction: column;
}
.card-footer {
  margin-top: auto;  /* empurra o rodapé para o fundo do card */
}

/* Layout principal com sidebar */
.layout {
  display: flex;
  gap: 24px;
}
.conteudo { flex: 3; }
.sidebar  { flex: 1; }
```

## CSS Grid {#sec-05-grid}

### O modelo de grelha bidimensional {#sec-05-grid-modelo}

O **CSS Grid** é um sistema de layout bidimensional — posiciona elementos
ao longo de **dois eixos simultâneos**: linhas e colunas. É optimizado para:

- Layouts de página completos (header, nav, main, sidebar, footer)
- Grelhas de conteúdo regulares (galeria de imagens, lista de produtos)
- Posicionamento preciso de elementos em relação a uma grelha

```css
/* Activar grid no contentor */
.contentor { display: grid; }
```

O conceito central é a **grelha explícita** — o programador declara quantas
colunas e linhas existem, e o browser coloca os filhos segundo essas regras.

### Definir colunas e linhas {#sec-05-grid-dimensoes}

As colunas e linhas são definidas pelas propriedades
`grid-template-columns` e `grid-template-rows`:

```css
.grelha {
  display: grid;

  /* 3 colunas: 200px fixo, 1fr flexível, 1fr flexível */
  grid-template-columns: 200px 1fr 1fr;

  /* 2 linhas automáticas */
  grid-template-rows: auto;

  /* Espaço entre células */
  gap: 16px;
}

/* A unidade fr (fraction) distribui o espaço disponível proporcionalmente */
/* 1fr + 2fr + 1fr = 4 partes: 25% + 50% + 25% */
.grelha-flex {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr;
  gap: 12px;
}
```

A função `repeat()` evita repetição para grelhas regulares:

```css
/* 12 colunas iguais */
.grelha-12 {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  gap: 16px;
}

/* Colunas responsivas automáticas: mínimo 200px, máximo 1fr */
.grelha-auto {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  gap: 16px;
}
```

### Posicionamento de elementos {#sec-05-grid-posicionamento}

Por omissão, o CSS Grid coloca os filhos sequencialmente, célula a célula.
Para controlo preciso, podem-se usar as propriedades de posicionamento:

```css
/* Notação: linha-início / linha-fim */
.item-a {
  grid-column: 1 / 3;   /* ocupa colunas 1 e 2 */
  grid-row: 1 / 2;      /* ocupa a linha 1 */
}

/* Alternativa com span: ocupa N colunas a partir da posição actual */
.item-b {
  grid-column: span 2;  /* ocupa 2 colunas */
}

/* Posicionamento absoluto na grelha */
.destaque {
  grid-column: 2 / 4;
  grid-row: 1 / 3;
}
```

### Template areas {#sec-05-grid-areas}

A propriedade `grid-template-areas` permite nomear zonas da grelha com
texto, tornando o layout visualmente declarativo:

```css
.pagina {
  display: grid;
  grid-template-columns: 220px 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header  header"
    "sidebar main"
    "footer  footer";
  min-height: 100vh;
  gap: 0;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }
```

O mapa textual de `grid-template-areas` reproduz visualmente o layout
pretendido. Um ponto (`.`) representa uma célula vazia.

### Flexbox vs CSS Grid — quando usar cada um {#sec-05-flex-vs-grid}

A escolha entre Flexbox e Grid não é de exclusão mútua — os dois sistemas
coexistem e complementam-se numa mesma página:

| | Flexbox | CSS Grid |
|:--|:--|:--|
| Dimensionalidade | Unidimensional (1 eixo) | Bidimensional (2 eixos) |
| Controlo | A partir do **filho** (tamanho flexível) | A partir do **contentor** (grelha explícita) |
| Caso de uso ideal | Distribuição de items em linha/coluna | Layouts de página, grelhas regulares |
| Alinhamento | Excelente num eixo | Excelente nos dois eixos |
| Reordenação | `order` por item | Posicionamento total |
| Suporte browsers | Excelente (IE 11+) | Excelente (Chrome 57+, 2017) |

**Regra prática:**

- Usar **Flexbox** para componentes — navbar, cards em linha, botões alinhados,
  centrar um elemento.
- Usar **Grid** para a estrutura da página — header/main/sidebar/footer,
  galerias de imagens, layouts complexos.

```css
/* Exemplo realista: Grid para a página, Flex para os componentes */

/* Estrutura da página — Grid */
.pagina {
  display: grid;
  grid-template-areas:
    "header"
    "main"
    "footer";
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}

/* Navbar dentro do header — Flex */
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 0 24px;
}

/* Grelha de cards dentro do main — Grid */
.galeria {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: 24px;
  padding: 24px;
}

/* Conteúdo interno de cada card — Flex */
.card {
  display: flex;
  flex-direction: column;
}
.card-footer { margin-top: auto; }
```

## Síntese do Capítulo {#sec-05-sintese}

O **box model** é o fundamento de todo o layout CSS: cada elemento ocupa
um rectângulo com quatro camadas — conteúdo, padding, border e margin.
A propriedade `box-sizing: border-box` torna o comportamento intuitivo e
é adoptada por todos os frameworks modernos.

O **float** foi o mecanismo de layout dominante durante a primeira décade
da web responsiva. A grelha manual de 12 colunas, construída com floats
e percentagens, demonstra os princípios que o Bootstrap depois automatizou.
As suas limitações — colapso do contentor, impossibilidade de alinhamento
vertical, complexidade responsiva — motivaram os sistemas modernos.

O **Flexbox** resolve o layout unidimensional: distribui elementos ao longo
de uma linha ou coluna, alinha-os nos dois eixos e adapta os seus tamanhos
ao espaço disponível. É o sistema adequado para componentes — navbars, filas
de cards, centração de elementos.

O **CSS Grid** resolve o layout bidimensional: posiciona elementos em relação
a uma grelha de linhas e colunas definida explicitamente no contentor.
É o sistema adequado para a estrutura da página e para grelhas regulares.
Os dois sistemas não são concorrentes — coexistem e complementam-se num
mesmo projecto.

## Exercícios {#sec-05-exercicios}

::: {.callout-caution}
#### Exercício 5.1 — Box model e box-sizing
Dado o seguinte CSS em modo `content-box`:

```css
.caixa {
  width: 300px;
  padding: 20px;
  border: 3px solid #333;
  margin: 16px;
}
```

a) Qual é a largura total ocupada pelo elemento (incluindo border)?
b) Qual seria a largura total em modo `border-box`?
c) Escreva a regra global que activa `border-box` em todos os elementos.

*Verificação:* a) 346px; b) 300px; c) `*, *::before, *::after { box-sizing: border-box; }`
:::

::: {.callout-caution}
#### Exercício 5.2 — Grelha de 12 colunas com floats
Construa, apenas com CSS e a técnica de floats, um layout com:

- Uma linha com três colunas de igual largura (4 + 4 + 4 colunas)
- Uma linha com uma coluna de conteúdo principal e uma sidebar (8 + 4 colunas)

Inclua o clearfix necessário para que o contentor não colapse.
:::

::: {.callout-caution}
#### Exercício 5.3 — Flexbox
Usando apenas Flexbox (sem Grid, sem floats), implemente:

a) Um contentor que centra um único elemento filho horizontal e verticalmente.
b) Uma barra de navegação com o logótipo à esquerda e quatro links à direita,
   todos alinhados verticalmente ao centro.
c) Uma linha de quatro cards que ocupa toda a largura disponível, com espaço
   igual entre os cards mas sem espaço nas extremidades.
:::

::: {.callout-caution}
#### Exercício 5.4 — CSS Grid
Usando CSS Grid e `grid-template-areas`, implemente o seguinte layout de página:

```
+-----------------------+
|        header         |
+-----------+-----------+
|  sidebar  |   main    |
+-----------+-----------+
|        footer         |
+-----------------------+
```

Condições: header e footer com altura fixa de 60px; sidebar com largura fixa
de 240px; main ocupa o espaço restante; a página ocupa sempre 100% da altura
do viewport.
:::

::: {.callout-caution}
#### Exercício 5.5 — Escolha do sistema de layout
Para cada situação, indique se usaria **Flexbox**, **Grid** ou **ambos**,
e justifique:

a) Distribuir cinco ícones de redes sociais em linha, com espaçamento igual.
b) Construir o layout completo de uma página com header, dois sidebars e área central.
c) Criar uma galeria fotográfica responsiva com colunas que se adaptam automaticamente.
d) Alinhar verticalmente o texto e o ícone dentro de um botão.
:::
